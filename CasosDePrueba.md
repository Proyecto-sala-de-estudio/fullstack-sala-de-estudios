# Casos de Prueba – US-03

| ID    | Qué se debe hacer (acción / entrada)        | Salida esperada                |
|-------|---------------------------------------------|--------------------------------|
| CP-01 | **Crear reserva exitosa en horario disponible:**<br>1. Navegar a `http://localhost:3001`.<br>2. Seleccionar la "Sala A" (salaId: 1).<br>3. Rellenar el formulario con:<br> - Estudiante: "Juan Pérez"<br> - Fecha: "2026-06-25"<br> - Hora: "10:00"<br>4. Hacer clic en "Reservar". | - El backend responde con estado `201 Created` y los datos del registro guardado en la base de datos.<br>- El frontend muestra una notificación de éxito.<br>- La reserva se añade automáticamente a la lista de reservas activas en pantalla. |
| CP-02 | **Intentar reservar sala en horario ocupado (solapamiento):**<br>1. Con la reserva de la "Sala A" para la fecha "2026-06-25" y hora "10:00" ya existente (CP-01).<br>2. Intentar hacer otra reserva para la misma "Sala A", la misma fecha ("2026-06-25") y la misma hora ("10:00").<br>3. Rellenar el formulario con:<br> - Estudiante: "María Gómez"<br>4. Hacer clic en "Reservar". | - El backend responde con estado `400 Bad Request` indicando el conflicto de horario.<br>- El frontend muestra un mensaje de alerta de error explicativo ("La sala ya cuenta con una reserva activa...").<br>- No se duplica la reserva ni se modifica la base de datos. |
| CP-03 | **Cancelar una reserva activa:**<br>1. Localizar la reserva de "Juan Pérez" (creada en CP-01) en la lista de reservas activas.<br>2. Hacer clic en el botón "Cancelar" asociado a esa reserva. | - El backend ejecuta la eliminación del registro de reserva mediante una petición `DELETE /api/reservas/:id` y responde con estado `200 OK`.<br>- El frontend elimina de inmediato la reserva de la lista visual.<br>- El horario de la sala queda liberado para nuevas reservas. |
