## Context

El README.md actual debe actualizarse para incorporar la documentacion del nuevo diseno del proyecto, los endpoints de salas, reservas y cursos, la guia detallada de construccion paso a paso y la metodologia de gestion de la configuracion (SWEBOK y GitHub Flow), removiendo todo tipo de emojis para mantener el tono formal.

## Goals / Non-Goals

**Goals:**
- Unificar la documentacion de todas las APIs (salas, reservas, cursos) en el README.md.
- Eliminar por completo los emojis para mayor seriedad profesional.
- Detallar las responsabilidades del equipo con precision basadas en la evidencia de Git y GitHub CLI.
- Mapear el flujo GitHub Flow aplicado en el repositorio a los conceptos de SWEBOK Cap. 8.

**Non-Goals:**
- Modificar el codigo funcional de la API de salas de estudio o cursos.
- Cambiar la base de datos o esquemas de tablas existentes.

## Decisions

- **Decision 1: Formato sobrio y profesional sin emojis.**
  - Razón: Se solicita expresamente eliminar emojis para mayor seriedad del proceso academico.
- **Decision 2: Estructura del README en base a la plantilla.**
  - Razón: Mantener la consistencia con la estructura de la guia de construccion de la API y el mapeo de gestion de la configuracion.
- **Decision 3: Mapeo de responsabilidades exactas usando evidencias de Git y GH CLI.**
  - Razón: Garantizar que la documentacion refleja con total veracidad el trabajo realizado por cada integrante.

## Risks / Trade-offs

- **Riesgo:** Perder consistencia en referencias historicas.
  - Mitigacion: Se mantendran los enlaces correctos a issues y pull requests reales del repositorio (como #11, #12, #13, #1, #3 y #5).
