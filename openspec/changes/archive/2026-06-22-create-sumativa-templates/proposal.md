## Why

El proyecto requiere estructurar y entregar la documentación formal obligatoria descrita en la pauta "EvaluacionSumativa2_Entrega3.md" utilizando las plantillas provistas para cumplir de manera estricta con la rúbrica de la Entrega 3 del proyecto. Esto incluye simplificar y adaptar el `README.md` raíz, y crear los archivos `EspecificacionHU.md`, `CasosDePrueba.md` y `DeudaTecnica.md` en el directorio raíz.

## What Changes

- Rediseñar el `README.md` principal del repositorio para ajustarse de forma estricta a la estructura y apartados definidos en la plantilla 9.1 de la pauta.
- Crear el archivo `EspecificacionHU.md` en la raíz del proyecto para documentar la historia de usuario `US-03: Reservar sala` de acuerdo con la plantilla 9.2.
- Crear el archivo `CasosDePrueba.md` en la raíz del proyecto conteniendo al menos 3 casos de prueba estructurados para la HU03 de acuerdo con la plantilla 9.3.
- Crear el archivo `DeudaTecnica.md` en la raíz del proyecto para documentar deudas técnicas, code smells y propuestas de mejora futuras de acuerdo con la plantilla 9.4.

- **Evidencia del Proyecto:** Entrega limpia de los artefactos requeridos para la corrección automatizada y manual del docente.
